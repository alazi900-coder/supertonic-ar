# سوبرتونيك ٣ — نسخة ويب عربية (متوافقة مع Android)

نسخة ويب عربية بالكامل (RTL) من العرض التوضيحي لـ Supertonic 3 لتحويل النص
إلى كلام، مصمَّمة لتعمل بسلاسة على متصفحات Android (Chrome, Samsung Internet,
Firefox) ولتكون قابلة للتثبيت على الشاشة الرئيسية كتطبيق ويب تقدمي (PWA).

## 🌍 تجربة مباشرة على Android

النسخة منشورة وتعمل بالكامل من المتصفح:
<https://dist-kzgljffo.devinapps.com/>

افتح الرابط من Chrome على Android، انتظر تحميل النماذج (≈٤٠٠ ميجا لمرة واحدة)،
ثم استخدم القائمة (⋮) ← **إضافة إلى الشاشة الرئيسية** لتثبيتها كتطبيق.

النماذج تُحمَّل ديناميكياً من `raw.githubusercontent.com` على الفرع
[`models`](https://github.com/alazi900-coder/supertonic-ar/tree/models) (مع
تقسيم `vector_estimator.onnx` إلى ٤ شرائح ودمجها في المتصفح لتجاوز حد ١٠٠ ميجا
لكل ملف على GitHub).

### 📦 استيراد النماذج من ملف ZIP (للاستخدام بدون إنترنت)

عند فتح التطبيق ستجد خيارين لتحميل النماذج:

1. **"⬇️ تنزيل من الإنترنت"** — السلوك القديم؛ يُنزّل النماذج كلها من
   GitHub مباشرة (≈ ٣٨٢ ميجا لمرة واحدة).
2. **"📦 استيراد ملف ZIP من جهازي"** — تختار ملف
   [`supertonic-ar-models.zip`](https://github.com/alazi900-coder/supertonic-ar/releases/download/models-v1/supertonic-ar-models.zip)
   من تخزين الهاتف (تكون قد نزّلته مسبقاً)، فيقوم المتصفح بفكّ ضغطه محلياً
   وتشغيل التطبيق فوراً بدون أي اتصال بالإنترنت. يفيد:
   - نقل المشروع بين أجهزة بدون شبكة جيدة.
   - الاحتفاظ بنسخة احتياطية.
   - تشغيل التطبيق في بيئات معزولة.

الملف الواحد يحتوي على كل نماذج ONNX + `tts.json` + `unicode_indexer.json` +
مجلد `voice_styles/` (١٠ أصوات).

> ⚠️ هذه نسخة من واجهة المستخدم فقط — النموذج وملفات الأصوات
> (`assets/onnx/`, `assets/voice_styles/`) هي نفسها المستخدمة في
> النسخة الإنجليزية الأصلية داخل مجلد [`../web/`](../web/README.md).

## ✨ المميزات

- 🌐 يعمل بالكامل داخل المتصفح، بدون أي خادم.
- 🚀 يدعم WebGPU مع رجوع تلقائي إلى WebAssembly عند عدم توفّره.
- 🗣️ ٣١ لغة مدعومة، مع تعيين العربية كلغة افتراضية.
- 🇸🇦 واجهة عربية كاملة باتجاه RTL وأرقام عربية-هندية للإحصاءات.
- 📱 تخطيط متجاوب مُحسَّن لشاشات Android، مع أزرار كبيرة مناسبة للمس.
- 📲 قابل للتثبيت على الشاشة الرئيسية كـ PWA عبر `manifest.webmanifest`.
- 💾 تنزيل الصوت المُولَّد كملف WAV.

## 📦 المتطلبات

- Node.js للحصول على خادم التطوير.
- متصفح حديث (Chrome 113+ مُفضَّل لتفعيل WebGPU).
- ملفات نموذج ONNX داخل `../assets/onnx/` وملفات أنماط الصوت داخل
  `../assets/voice_styles/`. راجع
  [`../web/README.md`](../web/README.md) لتفاصيل تنزيلها من Hugging Face.

## 🚀 التشغيل

```bash
cd web-ar
npm install
npm run dev
```

ثم افتح الرابط الذي يطبعه Vite (الافتراضي
[`http://localhost:3001`](http://localhost:3001)).

> **ملاحظة:** هذه النسخة تستخدم المنفذ `3001` بدلاً من `3000` لتتمكن من
> تشغيل النسختين الإنجليزية والعربية جنباً إلى جنب أثناء التطوير.

### تثبيت التطبيق على Android

1. افتح الصفحة في Chrome على هاتف Android.
2. اضغط على قائمة "⋮" ثم اختر **"إضافة إلى الشاشة الرئيسية"** أو
   **"تثبيت التطبيق"**.
3. سيظهر "سوبرتونيك" كأيقونة مستقلة على شاشتك الرئيسية، ويفتح
   التطبيق بدون شريط عنوان المتصفح.

## 🧭 الاستخدام

1. انتظر اكتمال تحميل النماذج (يظهر شريط حالة أعلى الصفحة).
2. اختر **نمط الصوت** (٥ ذكور + ٥ إناث).
3. اختر **لغة النص** (العربية افتراضياً).
4. اكتب أو الصق النص في خانة الإدخال.
5. اضبط **عدد الخطوات** و**السرعة** عند الحاجة.
6. اضغط **"🔊 توليد الصوت"**، ثم استمع للناتج أو نزّله كملف WAV.

## 🛠️ بنية الملفات

```
web-ar/
├── index.html              ← الواجهة العربية باتجاه RTL
├── style.css               ← تخطيط متجاوب مع تحسينات Android
├── main.js                 ← منطق UI ورسائل عربية
├── helper.js               ← منطق ONNX المشترك (مطابق لـ ../web/helper.js)
├── manifest.webmanifest    ← بيانات PWA
├── icon.svg                ← أيقونة التطبيق
├── sw.js                   ← Service worker مبسّط لجعل التطبيق قابلاً للتثبيت
├── vite.config.js          ← إعدادات Vite (port 3001)
└── package.json
```

> `helper.js` نسخة مطابقة من النسخة الإنجليزية في `../web/helper.js`
> ليبقى المشروع مكتفياً ذاتياً. عند تحديث النسخة الإنجليزية، يُنصح
> بمزامنته يدوياً.

## 🐞 استكشاف الأخطاء

- **لا يتم تحميل النماذج:** تأكد من وجود ملفات ONNX داخل `../assets/onnx/`.
- **WebGPU غير متاح:** سيرجع التطبيق تلقائياً إلى WebAssembly — هذا طبيعي
  على أجهزة Android القديمة.
- **النص يظهر بالعكس:** تأكد من أن المتصفح يدعم `dir="rtl"` (كل المتصفحات
  الحديثة تدعمه).
